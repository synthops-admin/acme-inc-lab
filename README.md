# ACME Inc Lab Template

**⚠️ This is a template repository. Do not use directly - fork this repository to create a new cohort repository.**

Welcome to ACME Inc monorepo template. This template provides the basic scaffolding that cohort members will build upon as they learn DevOps practices.

---

## 🏢 Meet the Teams

### **The 'Dev' Team**
Directly responsible for the company's core product - **The "ACME Inc" Web Application**.

The application is to be treated as a **production-grade application** for the purposes of this training.

All of the team's code lives in `dev/` directory.

### **The 'DevOps' Team**
This team is responsible for setting up and running all of ACME Inc's infrastructure.

The `dev` team relies on the `devops` team to provide them with a reliable and scalable infrastructure to deploy and run their application.

---

## 📁 Repository Structure

The repository is organized into the following directories:

| Directory | Purpose | Status |
|-----------|---------|--------|
| `dev/` | Contains all of dev team's code | 🚧 This is maintained by the dev team |
| `platform/` | Contains platform infrastructure configurations | 🚧 To be populated by cohort |
| `ci/` | Contains Continuous Integration configurations | 🚧 To be populated by cohort |
| `cd/` | Contains Continuous Deployment configurations | 🚧 To be populated by cohort |
| `monitoring/` | Contains monitoring configurations | 🚧 To be populated by cohort |

---

## 🚀 How to Use This Template

### For Cohort Creation:

1. **Fork this repository** to create a new cohort repository
2. **Rename the fork** to `acme-inc-cohort-{number}` (e.g., `acme-inc-cohort-2`)
3. **Update the README.md** to reflect the specific cohort information
4. **Clone the new repository** for cohort members to work with

### For Cohort Members:

1. Clone your cohort's repository
2. Navigate to your team's directory (`dev/` or `platform/`)
3. Follow the weekly tasks and assignments
4. Commit your changes and create pull requests for review
